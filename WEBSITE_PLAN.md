# T&A Contracting Professional Website & Client Portal Plan

## 1. Project Vision

T&A Contracting needs a professional business website and full operating portal that presents the company as an Arizona-native contracting and maintenance provider that is more capable, organized, and professional than a basic handyman service.

The finished platform should include:

- A public marketing website for leads and credibility.
- A client portal for requesting jobs, reviewing quotes, accepting work, paying invoices, and tracking job status.
- An admin/worker portal for managing clients, jobs, quotes, schedules, documents, payments, and communication.
- Automated email notifications and PDF quote delivery.
- A scalable back end that can later support online booking, recurring maintenance plans, reviews, service-area pages, and integrations.

Core brand message:

> T&A Contracting — Arizona-native maintenance, repairs, and improvements done right.

## 5. Client Portal Plan

The client portal should make T&A Contracting feel easier and more professional than typical service companies. Clients should be able to manage the full lifecycle of their jobs online.

### Client account features

- Secure client registration and login
- Password reset
- Client profile with name, email, phone, and saved addresses
- Multiple property addresses per client
- Communication preferences
- Job history
- Quote history
- Invoice and payment history

### Client dashboard

The client dashboard should show:

- Open job requests
- Quotes waiting for review
- Accepted quotes
- Scheduled jobs
- Completed jobs
- Unpaid invoices
- Recent messages or updates
- Quick button to request a new job

### New job request workflow

Clients should be able to submit a new job request with:

- Property address
- Service category
- Desired timeframe
- Description of work needed
- Priority level
- Photos or files
- Preferred contact method
- Access notes
- Special instructions

After submission, the system should:

- Create a job request record
- Notify admins by email
- Send the client a confirmation email
- Show the request in the client dashboard
- Allow admins to review, request more information, or create a quote

### Quote review workflow

Clients should be able to:

- View quote details online
- Download quote as a PDF
- See itemized labor, materials, fees, discounts, taxes, and total
- See expiration date
- See included and excluded work
- Ask questions about the quote
- Accept the quote electronically
- Decline the quote with an optional reason
- Request changes

### Quote acceptance workflow

When a client accepts a quote, the system should:

- Capture acceptance timestamp
- Record the accepted quote version
- Require terms acknowledgment
- Notify admins
- Convert the quote into an approved job
- Optionally collect deposit payment
- Show next steps to the client

### Client payment workflow

Clients should be able to:

- Pay deposits
- Pay final invoices
- View payment status
- Download receipts
- Store payment method if supported by the payment provider
- See invoice due dates

Recommended payment provider:

- Stripe for card and ACH payments

### Client communication features

- Job-specific message thread
- Email notifications for important updates
- Quote and invoice email delivery
- Optional SMS notifications later
- Internal/admin-only notes hidden from clients

## 7. Quote, PDF & Email Plan

Quotes are a major part of this platform and should feel polished and professional.

### Quote builder features

Admins should be able to create itemized quotes with:

- Client and property details
- Scope of work
- Line items
- Labor
- Materials
- Fees
- Discounts
- Taxes, if applicable
- Deposit requirement
- Total price
- Quote expiration date
- Terms and conditions
- Internal notes
- Attachments or reference photos

### Quote versioning

The system should store quote versions so that accepted quotes are preserved exactly as the client accepted them.

### Quote PDF

Each quote should generate a professional PDF that includes:

- T&A Contracting logo
- Business contact information
- Quote number
- Quote date
- Expiration date
- Client information
- Job/property address
- Scope of work
- Itemized costs
- Total amount
- Deposit requirement, if any
- Terms and conditions
- Acceptance instructions

### Quote email workflow

When an admin sends a quote, the system should:

- Generate the PDF
- Email the client with a quote summary
- Attach the PDF or provide a secure download link
- Include a secure button/link to review, accept, decline, or ask questions
- Track email sent time
- Mark quote status as “sent”

### Email templates

Recommended email templates:

- New job request confirmation
- Admin notification for new request
- Quote sent
- Quote accepted confirmation
- Quote declined notification
- Deposit payment request
- Payment receipt
- Job scheduled
- Job reminder
- Job completed
- Invoice sent
- Password reset

## 8. Payments & Invoicing Plan

The payment system should make it easy for clients to approve work and pay online.

### Recommended features

- Stripe integration
- Deposit payments
- Final invoice payments
- Payment status tracking
- Paid/unpaid/partial/refunded statuses
- Automatic receipt emails
- Admin payment dashboard
- Invoice PDF generation
- Invoice email delivery

### Invoice workflow

1. Quote is accepted.
2. Deposit invoice is generated if required.
3. Client pays deposit online.
4. Admin schedules job.
5. Job is completed.
6. Final invoice is generated.
7. Client pays final balance.
8. Receipt is sent.
9. Job is closed.
