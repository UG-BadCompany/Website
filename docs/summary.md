# T&A Contracting Portal Summary

_Last updated: 2026-05-13_

## Current production foundation

- Static site output is generated by `scripts/build-static-site.mjs` and verified by `scripts/ensure-netlify-out.mjs`; Netlify publishes the `out/` directory.
- Server-side behavior is implemented with Netlify Functions under `netlify/functions/`.
- Netlify Database migrations live under `netlify/database/migrations/` and are checked before builds by `scripts/prebuild-netlify-migrations.cjs`.
- Passwordless magic-link auth stores hashed magic-link tokens and hashed session tokens only.

## Authentication and sessions

- Auth uses a single `ta_session` cookie by default.
- Session lifetimes are role-aware:
  - Clients: `CLIENT_SESSION_TTL_MINUTES=30`.
  - Staff/admin users: `STAFF_SESSION_TTL_MINUTES=120`.
- Session cookies include `Max-Age` and `Expires`.
- `/api/me` refreshes `auth_sessions.last_seen_at`, extends `auth_sessions.expires_at`, and returns a refreshed session cookie.
- Magic-link verification computes the session TTL after loading the user's assigned roles.
- Magic-link verification now uses the original redirect-with-Set-Cookie flow so the browser lands on `/dashboard/` with the session cookie set directly on the verification response.

## Roles and permissions

- Built-in roles are `client`, `worker`, and `admin`.
- Client defaults include dashboard, request, quote, and invoice permissions.
- Admin defaults include every registered permission, including user, role, request, quote, invoice, and dashboard switching permissions.
- `/api/me` returns permission booleans for dashboard tools, user/role management, request/quote management, invoice viewing/management, and dashboard view switching.

## Implemented API areas

- Auth/profile:
  - `POST /api/auth/request-magic-link`
  - `GET /api/auth/verify`
  - `POST /api/auth/logout`
  - `GET/PATCH /api/me`
- Client workflows:
  - `GET/POST /api/client/job-requests`
  - `GET/PATCH /api/client/quotes`
  - `GET /api/client/invoices`
- Worker workflows:
  - `GET/PATCH /api/worker/jobs`
- Admin workflows:
  - `GET/PATCH /api/admin/job-requests`
  - `GET/POST/PATCH /api/admin/quotes`
  - `GET/PATCH /api/admin/invoices`
  - `GET/POST/PATCH /api/admin/users`
  - `GET/POST/PATCH /api/admin/roles`

## Database and migration notes

- Migrations through `0014` cover the current portal, schedule, roles/permissions, assignments, completion review, quote/payment controls, invoices/payments, worker details, and completion evidence.
- Historical Netlify-applied migration filenames are preserved when required. Compatibility migrations that have already been applied must keep their original SQL bodies.
- The prebuild guard removes known stale cached migration names only when a later replacement is present, then checks migration filename prefixes while allowing known compatibility names.

## Payment and invoice status

- Invoice and payment tables are available for manual/admin-confirmed payment tracking.
- Clients can view their unpaid invoices through `/api/client/invoices`.
- Admins can list unpaid invoices and confirm payments through `/api/admin/invoices`; payment confirmation records a payment, marks the invoice paid, completes the linked job request, and writes an audit event.
- Square/hosted checkout integration is not yet implemented in the current codebase.

## Recent auth repair

- Reverted the magic-link verification handoff page back to a direct `302` dashboard redirect with `Set-Cookie`, matching the login flow that previously worked reliably.
- Added a regression check that the login page declares its query-status helpers only once, preventing duplicate inline script declarations around magic-link status messages.
- Changed the admin invoice action binder from a top-level `const` expression to a redeclaration-safe function declaration to avoid the browser error `Identifier 'bindAdminInvoiceActions' has already been declared`.
